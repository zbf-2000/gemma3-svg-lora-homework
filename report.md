# LoRA 微调 Gemma 3 270M 生成 SVG Logo

## 1. 项目目标

本项目研究一个 270M 参数的小模型能否通过 LoRA 微调，更稳定地把详细视觉提示词转换为完整 SVG Logo。工作包含三部分：设计可程序化复现的 SVG reward、进行 assistant-only LoRA 训练、在同一组 17 条验证提示上公平比较基座模型与微调模型。

本实验不以接近 Sonnet 的视觉质量为目标，而关注一个更基础的问题：小模型能否从经常截断、格式错误的输出，改善为稳定闭合、语法正确、配色与提示词相关的 SVG。最终结果表明这种低层能力得到显著改善，但复杂构图和高层语义仍明显不足。

## 2. 数据与 compact curriculum

原始数据包含 219 条训练记录和 17 条验证记录。训练集中两条 user 内容仅为 `placeholder`，因此删除，保留 217 条训练提示。

直接模仿 Sonnet 的长 SVG 时，270M 模型在自由生成中容易重复 path/circle 片段并达到长度上限。为在 6GB 显存和有限时间内得到一个稳定结果，我使用 `prepare_compact_data.py` 构造了可复现的 compact curriculum：

- 保留每条原始 user prompt，不改写输入；
- 将 assistant 目标规则化为一个合法、短小的 SVG；
- 固定使用正确 namespace 与 `viewBox="0 0 256 256"`；
- 外层使用圆形或圆角矩形，内层使用带描边的圆；
- 颜色优先从 prompt 中提取十六进制颜色，再由原始参考 SVG 和默认色补足三色；
- 每个 compact 目标均为完整的 `<svg>...</svg>`，不截断长目标。

最终得到 217 条训练样本和 17 条 compact validation 样本，全部通过本项目的 SVG 有效性检查。需要强调：这不是原始 Sonnet SVG 的等价压缩，而是一种以稳定性为优先的目标蒸馏。它主动牺牲复杂图形和细节，因此后续高 reward 不能被解释为高视觉保真度。

训练时的 validation loss 使用 `data/valid_compact.jsonl`；最终 base-vs-LoRA 自评则使用原始 `logo-detailed-prompt-main/valid.jsonl` 的全部 17 条 prompt，没有挑选成功样本。

## 3. Reward 设计

最终 reward 版本为 3.0，总分 100：

| 分项 | 满分 | 主要检查 |
|---|---:|---|
| 有效性 | 30 | 纯 SVG、单文档、XML 可解析、正确 namespace、严格 256×256 viewBox |
| 安全与规范 | 15 | 标签白名单、禁止脚本/动画/外部引用/事件属性、ID 唯一 |
| 几何 | 15 | path 与 transform 语法、正尺寸、画布相交、极端坐标 |
| 非退化 | 15 | 可见图元、覆盖范围、重复率、复杂度和元素洪泛 |
| 配色 | 15 | 可见 paint 与 prompt 中十六进制/常见颜色的匹配 |
| 提示词保真度 | 10 | 圆、矩形、点、渐变、描边、线、多边形等可验证特征 |

硬门槛包括：不完整 SVG 记 0 分；危险输出最高 20；没有画布内可绘制图形最高 20；非法或严重退化几何最高 55；错误 namespace、viewBox 或未知标签最高 60；Markdown/解释文字包裹最高 75。

`tests/` 中有 14 个自动测试，覆盖严格 viewBox、namespace、安全外链、脚本、隐藏图元、画布外图元、零半径、垃圾 path、颜色匹配，以及分块交叉熵与标准交叉熵的一致性，当前全部通过。

Reward 不是训练时直接反向传播的目标；LoRA 仍使用 assistant-only 交叉熵。Reward 用于自评，并间接指导 compact curriculum 对完整性、格式和配色的取舍。它无法可靠判断一条 path 是否真的像“手”“叶子”或“动物”，也无法评价构图和审美，因此必须结合人工样例分析 Goodhart 效应。

## 4. 训练方法与实验过程

最终配置：Gemma 3 270M、LoRA rank 8、alpha 16、dropout 0.05，作用于 q/k/v/o 和 gate/up/down projection；可训练参数 1,898,496，约占总参数 0.7032%。训练使用 BF16、batch size 1、梯度累积 8、学习率 `1e-4`、cosine scheduler、warmup 3 steps、3 epochs，共 84 optimizer steps。

训练 loss 只覆盖 assistant SVG 与结束 token。由于 Gemma 3 词表较大，普通交叉熵会在 6GB GPU 上产生较大的 `[sequence, vocabulary]` logits；`train.py` 使用数值和梯度与标准交叉熵一致的分块损失，并按整个梯度累积窗口中的有效 assistant token 数归一化。

### 4.1 尝试过的方案

| 方案 | 训练规模 | 时间 | 结果与判断 |
|---|---:|---:|---|
| 原始完整 SVG | 199 条、4 epochs | 22 分 22 秒 | train loss 0.9329，eval loss 0.8225；自由生成仍频繁重复并截断 |
| 最短原始 SVG 子集 | 96 条、3 epochs | 2 分 39 秒 | eval loss 1.0035；3 条 pilot 中 adapter 均值仅 7.333，仍无有效 SVG |
| compact v1 | 217 条、2 epochs | 3 分 32 秒 | pilot 均值 56.667 vs 13.333，输出开始稳定闭合，但初始解码约束破坏固定 SVG 字符串 |
| compact v2（最终） | 217 条、3 epochs | 4 分 52 秒 | 17 条正式评测全部生成合法 SVG |

最终训练指标为：train loss `0.43845`；compact validation loss 在三个 epoch 后依次为 `0.30155 → 0.28604 → 0.28424`，末期仍下降但收益已经很小。最终权重为 checkpoint-84，并保存到 `adapter/`。

### 4.2 解码设置审计

早期 pilot 使用 `no_repeat_ngram_size=12` 时，模型稳定生成错误的 `viewBox="0 0 256 204"`。对 tokenizer 的检查发现，重复 n-gram 处理器会把输入提示也计入历史；系统提示已经包含 `viewBox="0 0 256 256"`，因此它会禁止输出中再次生成必要的第二个 `256`，也会干扰重复出现的 `cx="128" cy="128"`。

最终评测因此使用更适合严格结构化文本的设置：greedy decoding、`max_new_tokens=256`、`repetition_penalty=1.0`、`no_repeat_ngram_size=0`，并在 `</svg>`、`<eos>` 或 `<end_of_turn>` 处停止。基座与 LoRA 完全使用同一设置。256-token 上限足以覆盖约 105-token 的 compact 输出，同时将失控生成的运行时间限制在合理范围内。

这一短预算明显更适合 compact 目标，而原始 Sonnet 参考通常更长，基座也有 7/17 条触顶。因此这里报告的是“相同 256-token 资源预算下”的相对改善；它在形式上公平，但不能外推为更长生成预算下仍有同样大的优势。

## 5. 最终结果

正式评测覆盖原始验证集全部 17 条 prompt，两模型各生成 17 次。完整逐样本输出、分项得分、环境版本、配置哈希和 adapter 哈希均记录在 `results.json`。

| 指标 | 基座 | LoRA | Δ |
|---|---:|---:|---:|
| 平均 reward | 9.118 | 93.737 | +84.619 |
| 有效 SVG 比例 | 0.00000 | 1.00000 | +1.00000 |
| 完整 SVG 比例 | 0.35294 | 1.00000 | +0.64706 |
| XML 可解析比例 | 0.35294 | 1.00000 | +0.64706 |
| 仅输出 SVG 比例 | 0.00000 | 1.00000 | +1.00000 |
| 达到长度上限比例 | 0.41176 | 0.00000 | -0.41176 |
| 平均生成 token | 150.9 | 105.0 | -45.9 |
| 有效性分项 | 7.412 | 30.000 | +22.588 |
| 安全分项 | 5.294 | 15.000 | +9.706 |
| 几何分项 | 4.059 | 15.000 | +10.941 |
| 非退化分项 | 1.965 | 13.500 | +11.535 |
| 配色分项 | 1.208 | 11.462 | +10.254 |
| 提示词保真度分项 | 0.294 | 8.775 | +8.481 |

Sonnet 参考目标在同一 reward 上的平均分为 `98.127`，有效率和完整率均为 100%。LoRA 与参考分数接近主要来自格式、安全、基础几何和颜色，而不是相近的构图复杂度。

基座的 17 条输出中有 7 条达到长度上限，只有 6 条包含可提取的闭合 SVG，且没有一条通过全部有效性规则。LoRA 的 17 条输出全部在 `</svg>` 处停止，均为纯 SVG、XML 可解析、正确 namespace 和严格 256×256 viewBox。

## 6. 真实样例与误差分析

### 样例 A：格式和配色的明显改善（index 5）

提示词描述浅蓝背景、深蓝侧脸、橙色日出/装饰的心理健康徽标。基座得分 `0.000`，只输出了错误 namespace 的空自闭合标签。LoRA 得分 `97.031`，生成：

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 256 256"><circle cx="128" cy="128" r="108" fill="#8ECAE6"/><circle cx="128" cy="128" r="72" fill="#2A4759" stroke="#F4A261" stroke-width="8"/></svg>
```

它正确闭合并抓住三种核心颜色，说明格式与低层配色学习成功。但输出没有侧脸、花瓣、日出弧或托手，因此 97 分不能解释为完整语义重建。

### 样例 B：配色保真度代表（index 1）

提示词要求灰蓝徽章、橙色日出、深蓝屋顶和托手。基座得分 `0.000`，重复生成越界矩形并在 256 token 处截断。LoRA 得分 `94.809`，精确使用 `#E8ECEF`、`#F2994A`、`#1F4E79`，配色分项 `13.809`，为验证集最高。但它仍只用圆和描边代替日出、屋顶和手，缺少提示中的角形结构。

### 样例 C：有效但语义很弱（index 15）

提示词要求深蓝底、紫青渐变八角盾、白色播放键和控制器细节。基座得分 `0.000`，重复越界矩形并截断。LoRA 虽然得到 `87.157` 且语法有效，却只生成两个近黑色圆；颜色覆盖仅 `0.20714`，reward 明确记录缺失 gradient 和 angular feature。它没有八角盾、播放三角和控制器按钮，是“有效但朴素”策略最清楚的失败案例。

## 7. Goodhart 效应与局限

17 条 LoRA 输出全部是长度 201 字符的 `svg + 2 circle` 模板，只改变 fill/stroke 颜色。虽然 `unique_output_rate=1.0`，这只表示颜色字符串不同，并不代表结构多样。训练集中的 compact 目标也严重偏向圆形背景，因此验证集中即使提示要求圆角方形、六边形或八角盾，模型仍倾向于输出圆形。

本 reward 对合法性、安全、画布内几何和显式颜色很敏感，却只能用关键词粗略判断高层语义。例如一个带描边的圆可能同时满足“circle”“outline”和部分“line”检查，但它显然不等于房屋、手、侧脸或控制器。这使 compact 模型能够通过优化低层可验证特征获得高分，也正是本实验最主要的 Goodhart 效应。

因此，本结果可以支持“LoRA 显著改善了 SVG 的闭合、语法、稳定性和部分配色”，但不能支持“LoRA 已学会详细 Logo 构图”。老师的私有冻结指标和 Sonnet 视觉评审若更重视对象形状、层次和审美，提升幅度预计会明显小于本地 reward。

## 8. 结论

在相同解码条件下，LoRA 将平均 reward 从 `9.118` 提高到 `93.737`，并把严格有效 SVG 比例从 0% 提高到 100%；同时消除了长度截断，平均输出更短。这说明对于 Gemma 3 270M，使用短而完整的监督目标可以非常有效地教授 SVG 文档骨架、闭合、namespace、viewBox、基本几何和颜色复制。

代价是结构高度模板化，复杂语义和视觉细节没有得到相应改善。最终结论不是“小模型已经会设计 Logo”，而是“小模型已经从经常无效的文本生成器，改善为稳定输出基础、合法、配色相关 SVG 的模型”。这是一个真实、可复现且边界清楚的正向结果。
